## 2. KEY DISCOVERY: FLOW HAS EVM COMPATIBILITY (Crescendo Upgrade)

Since Sept 2024, Flow has FULL EVM equivalence via the "Crescendo" upgrade:
- Solidity contracts run natively on Flow EVM
- MetaMask connects to Flow EVM (chainId: 747)
- Uniswap V2/V3 forks exist (KittyPunch PunchSwap)

This means TWO possible approaches for Flow token swaps:
  A) Native Cadence path (FCL + Cadence scripts) — more complex, deeper Flow
  B) EVM path (ethers.js + Flow EVM RPC) — simpler, reuses existing EVM code

>>> RECOMMENDATION: Use the EVM path for swaps (simpler), FCL for NFT transfers

#### [NEW] flow_tools.py
All Flow-specific tool implementations:

  flow_get_available_tokens()
    - Query KittyPunch/IncrementFi API for available Flow tokens
    - Cache token list (similar to knowledge_base.py)

  flow_get_swap_quote(token_in, token_out, amount)
    - Call PunchSwap/IncrementFi router for quote
    - PunchSwap is Uniswap V2/V3 fork — use standard router ABI
    - Flow EVM RPC: https://mainnet.evm.nodes.onflow.org
    - Flow EVM chain ID: 747
    - Return: { amount_out, rate, path, router_address }

  flow_confirm_swap(quote)
    - Build the swap transaction (Cadence or EVM)
    - For EVM path: standard Uniswap V2 swapExactTokensForTokens call
    - Return transaction payload for frontend signing

  flow_get_user_nfts(account_address)
    - Query Flow NFT catalog API or Alchemy Flow API
    - Endpoint: https://flow-nft-catalog.com (community catalog)
    - Return: list of { id, name, collection, image, contract_address }

  flow_transfer_nft(nft_id, collection, to_address)
    - Build Cadence transaction for NFT transfer
    - Uses NonFungibleToken.transfer() standard
    - Cadence script:
        import NonFungibleToken from 0x1d7e57aa55817448
        transaction(recipient: Address, withdrawID: UInt64) {
          prepare(signer: auth(BorrowValue) &Account) {
            let collection = signer.storage
              .borrow<auth(NonFungibleToken.Withdraw) &{NonFungibleToken.Collection}>(
                from: /storage/[CollectionPath]
              )!
            let nft <- collection.withdraw(withdrawID: withdrawID)
            let recipientCollection = getAccount(recipient)
              .capabilities.get<&{NonFungibleToken.Collection}>(/public/[CollectionPath])
              .borrow()!
            recipientCollection.deposit(token: <-nft)
          }
        }
    - Return transaction payload for FCL signing

## 7. FLOW TOKEN SWAP APPROACH (via PunchSwap/Flow EVM)

Since Flow has EVM equivalence, we can use the SAME EVM signing code:

1. PunchSwap = Uniswap V2 fork on Flow EVM
   - Router: standard swapExactTokensForTokens ABI
   - Factory: standard Uniswap V2 factory ABI
   - Chain ID: 747
   - RPC: https://mainnet.evm.nodes.onflow.org

2. Quote flow:
   User says "swap 10 FLOW for USDC" →
   Backend calls PunchSwap router.getAmountsOut(amount, [FLOW, USDC]) →
   Returns quote to user →
   User confirms →
   Backend builds swapExactTokensForTokens tx →
   Frontend signs via FCL EVM or MetaMask →
   Done

3. Key tokens on Flow EVM:
   - FLOW (native, wrapped as WFLOW)
   - USDC, USDT (bridged)
   - stFLOW (liquid staking)
   - Various community tokens

## 8. NFT TRANSFER APPROACH

Flow NFTs use Cadence's NonFungibleToken standard:

1. User says "transfer my TopShot moment #12345 to 0xabc123"
2. Agent calls flow_get_user_nfts() to list user's NFTs
3. User selects the NFT
4. Agent calls flow_transfer_nft(nft_id, collection, recipient)
5. Backend builds Cadence transaction
6. Frontend signs via FCL (Flow Wallet / Dapper)
7. Transaction confirmed

Key NFT collections on Flow:
  - NBA Top Shot (topshot)
  - NFL ALL DAY
  - UFC Strike
  - CryptoKitties (original!)
  - .find (Flow name service)

API for listing user NFTs:
  - Flow NFT Catalog: https://flow-nft-catalog.com
  - Alchemy Flow API (if available)
  - Direct Cadence scripts querying account storage

## 11. IMPORTANT NOTES

- Flow addresses are 0x + 16 hex chars (NOT 40 like EVM)
  Example: 0x1654653399040a61
  The agent must validate Flow addresses differently from EVM!

- Flow EVM addresses ARE standard 0x + 40 hex chars
  Same as Ethereum, since Flow EVM is EVM-compatible

- User can connect BOTH HOT Kit AND Flow wallet simultaneously
  The frontend should detect which wallet is "active" for the current swap

- Flow Wallet and Dapper Wallet use FCL discovery protocol
  No need to import specific wallet SDKs — FCL handles wallet selection

- Flow EVM is a separate environment from Cadence
  Tokens on Flow EVM ≠ tokens on Cadence flow
  The Cadence bridge can move tokens between environments
